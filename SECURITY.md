# Security Policy

## Supported Versions
We provide security updates for the following versions of Dynamite Phishing:

| Version | Supported |
|---------|-----------|
| 1.0.x   | ✅         |
| 0.x     | ❌ (use latest release) |

## Reporting a Vulnerability
We take security seriously. If you discover a vulnerability:

1. **Do not open a public issue.**
2. Email the maintainers at: security@dynamitephishing.org (placeholder).
3. Include:
   - Steps to reproduce
   - Affected version(s)
   - Potential impact
   - Suggested fix if available

We will acknowledge receipt within **72 hours**, provide an initial assessment within **7 days**, and aim to release a fix within **30 days**.

## Responsible Disclosure
- Please avoid publishing exploit details until a fix is released.
- We credit reporters in release notes unless anonymity is requested.
- Critical vulnerabilities may be eligible for a bounty (see `BOUNTY.md`).

## Scope
- Code in this repository
- Build scripts and configuration
- Documentation related to security

Out of scope:
- Third‑party dependencies (report upstream)
- Social engineering or phishing attempts against maintainers
